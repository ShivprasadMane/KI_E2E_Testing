# KeyInvest E2E Tests

End-to-end tests for the KeyInvest portal on the **test environment**.

| | URL |
|---|-----|
| Portal | https://polite-plant-02b096d00.6.azurestaticapps.net |
| API | https://test-kiep-api.azurewebsites.net |

## Setup

```bash
cd e2e_Testing
cp .env.example .env
# Fill B2C test user credentials in .env
npm install
npm run install:browsers
npm run matrix:generate   # creates projects/keyinvest/data/matrix.xlsx
```

## Excel matrix (primary — login tests)

Edit `projects/keyinvest/data/matrix.xlsx` sheet **TestMatrix**:

| Column | Required | Notes |
|--------|----------|-------|
| caseNo | Yes | Unique id, e.g. `TC-01` (any label works — only used in reports) |
| Enabled | No | `Y` / `N` |
| Workflow | Yes | Named shortcuts: `login`, `dashboard`, `dashboard-full`, `clients-full` (funeral, adviser, investor), `policies-full` (funeral, adviser, investor, admin), `applications-full` (funeral, adviser, investor, admin), `funeral-full`, `dashboard-links` (funeral), `investor-links` (investor), `adviser-links`, `dashboard-exports`, `clients-table`, `policies-table`, `applications-table`, `portfolio-check`, `recent-app`, `summary-filters`. Full chains like `Login > Dashboard > …` still work. |
| Persona | Yes | `guest`, `investor`, `funeral`, `adviser`, `admin` |
| username | No | Empty → `.env` for that Persona |
| password | No | Empty → `.env` for that Persona |
| expectedResult | No | `success` (default) or `validation_error` |
| bondType | No | `NOMINATED` or `PREPAID` (create-application workflows) |
| dataRow | No | Key into sheet **CreateApplicationData** |
| scenario | No | Key into sheet **CreateApplicationValidation** |
| documentSet | No | PDF set under `projects/keyinvest/data/documents/create-application/` |

Additional Excel sheets (generated by `matrix:generate`):

- **CreateApplicationData** — wizard field input per `dataRow` (TMD answers, investor, amounts, payment, `documentSet`)
- **CreateApplicationValidation** — negative cases grouped by `scenarioId`

Create-application workflows: `create-application-full`, `create-application-tmd`, `create-application-validation` (TC-23+ in generated matrix; **Enabled=N** by default).

```bash
npm run validate:matrix          # preflight — catch typos before opening a browser
npm run matrix:plan              # list enabled tests from Excel
npm run test:matrix              # recommended — 1 login per persona, all rows in one go
npm run test:matrix:raw          # 1 login per Excel row (strict per-row login)
npm run test:matrix:funeral      # funeral rows only — saved session, no B2C
npm run test:matrix:headed       # watch in browser
npm run matrix:generate          # regenerate matrix.xlsx from scripts/generate-login-matrix.ts
npm run test:create-application  # Excel rows with create-application-* workflows (enable TC-23+ first)
npm run test:create-application:specs  # @create-application persona specs (TMD + entry smoke)
```

Place upload PDFs in `projects/keyinvest/data/documents/create-application/` (`default-id.pdf` is seeded from sample).

## Code-based tests (alternative)

```bash
npm run test:login               # all personas (hardcoded spec)
npm run test:login:funeral       # single persona
npm run test:smoke               # login page loads (no credentials)
npm run test:guest
npm run test:funeral:export      # funeral dashboard Excel + PDF per widget
npm run test:adviser:export      # adviser dashboard Excel + PDF per widget
```

## CI (GitHub Actions)

Workflow: `.github/workflows/e2e-matrix.yml`

| Job | What it runs |
|-----|----------------|
| `validate-matrix` | `matrix:generate` + `validate:matrix` |
| `matrix-grouped` | `npm run test:matrix` (Excel TC-01 … TC-22) |
| `create-application` | `npm run test:create-application` (optional; enable TC-23+ in Excel) |
| `dashboard-exports` | `test:funeral:export` + `test:adviser:export` (full Excel + PDF) |

**Repository secrets** (Settings → Secrets and variables → Actions):

- `INVESTOR_EMAIL`, `INVESTOR_PASSWORD`
- `FUNERAL_DIRECTOR_EMAIL`, `FUNERAL_DIRECTOR_PASSWORD`
- `FINANCIAL_ADVISER_EMAIL`, `FINANCIAL_ADVISER_PASSWORD`
- `ADMIN_EMAIL`, `ADMIN_PASSWORD`

Portal/API URLs default to the test environment in the workflow file. Override with `PORTAL_URL` / `API_URL` secrets if needed.

Triggers: push/PR to `main`, `master`, `dev`, `test`, and manual `workflow_dispatch`.
